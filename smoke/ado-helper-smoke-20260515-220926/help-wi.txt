Manage work items (alias: wi)

Usage: ado work-item [OPTIONS] <COMMAND>

Commands:
  create          Create a new work item
  list            List work items matching filters
  view            View details of a work item
  update          Update fields on a work item
  delete          Delete a work item (soft-delete to recycle bin by default)
  comment         Add a comment to a work item
  comments        List comments on a work item
  comment-edit    Edit an existing comment
  comment-delete  Delete a comment
  link            Add a relation (parent/child/related/hyperlink) to a work item
  links           List relations on a work item
  link-rm         Remove a relation by index (see `wi links` for indices)
  attach          Upload a file and attach it to a work item
  history         Show revision history of a work item
  open            Open a work item in the browser
  help            Print this message or the help of the given subcommand(s)

Options:
      --org <ORG>
          Override the organization URL from config (e.g. https://dev.azure.com/myorg)

      --project <PROJECT>
          Override the default project from config

      --output <OUTPUT>
          Output format

          Possible values:
          - text:  Plain text, one item per line (default)
          - json:  JSON — full API response, useful for scripting
          - table: Column-aligned table (only meaningful for list commands; falls back to text elsewhere)
          
          [default: text]

  -h, --help
          Print help (see a summary with '-h')
