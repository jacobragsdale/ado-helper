Edit title / description / arbitrary fields on a pull request

Usage: ado pr update [OPTIONS] <ID>

Arguments:
  <ID>
          Pull request ID

Options:
      --org <ORG>
          Override the organization URL from config (e.g. https://dev.azure.com/myorg)

      --title <TITLE>
          New title

      --description <DESCRIPTION>
          New description

      --project <PROJECT>
          Override the default project from config

      --field <NAME=VALUE>
          Generic field set, repeatable. Use either short alias or full ADO key. Examples: --field draft=false  --field status=active  --field autoCompleteSetBy=<id>

      --output <OUTPUT>
          Output format

          Possible values:
          - text:  Plain text, one item per line (default)
          - json:  JSON — full API response, useful for scripting
          - table: Column-aligned table (only meaningful for list commands; falls back to text elsewhere)
          
          [default: text]

      --repo <REPO>
          Repository name (defaults to ADO_REPO env var or origin remote)

  -h, --help
          Print help (see a summary with '-h')
