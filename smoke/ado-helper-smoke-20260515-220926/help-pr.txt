Manage pull requests

Usage: ado pr [OPTIONS] <COMMAND>

Commands:
  create          Create a new pull request
  list            List pull requests
  view            View details of a pull request
  update          Edit title / description / arbitrary fields on a pull request
  approve         Approve a pull request (vote = 10, configurable via --vote)
  comment         Post a top-level comment on a pull request
  threads         List comment threads on a pull request
  thread-reply    Reply to an existing pull request comment thread
  thread-resolve  Close an existing pull request comment thread
  complete        Complete (merge) a pull request
  abandon         Abandon a pull request (close without merging)
  reactivate      Reactivate an abandoned pull request
  open            Open a pull request in the browser
  help            Print this message or the help of the given subcommand(s)

Options:
      --org <ORG>
          Override the organization URL from config (e.g. https://dev.azure.com/myorg)

      --project <PROJECT>
          Override the default project from config

      --output <OUTPUT>
          Output format

          Possible values:
          - text:  Plain text, one item per line (default)
          - json:  JSON — full API response, useful for scripting
          - table: Column-aligned table (only meaningful for list commands; falls back to text elsewhere)
          
          [default: text]

  -h, --help
          Print help (see a summary with '-h')
